# Mission

## 目标

依据已评审通过的《LittleDuck 基于 LLM 的 H5 Chatbot MVP 产品需求文档》V1.7，完成 LittleDuck Chatbot MVP 的产品建设、测试和腾讯云部署。

最终应形成一个可通过公网访问的服务：

- 普通用户可在移动端浏览器使用 H5 完成注册、登录、LLM 多轮对话和历史会话访问；
- 管理员可在 PC Web 管理端配置 OpenAI API Key 和模型，并按话题查看聊天记录、每次 OpenAI 调用的 Prompt 和返回内容；
- 系统代码、协作文件、测试材料和部署文档保存在 GitHub 仓库 `wanyq/LittleDuck`；
- 系统部署在用户提供的腾讯云服务器上，并具备可重复执行的部署和更新方式。

## 范围

### 包含

- 以 `LittleDuck_Chatbot_MVP_PRD.md` V1.7 为产品需求和验收依据；
- 用户端 H5：
  - 手机号和固定验证码 `000000` 注册；
  - 手机号和固定验证码 `000000` 免密登录；
  - 登录态保持和退出登录；
  - 新对话、多轮纯文本聊天、流式回复；
  - 历史会话的展示、时间分组、标题搜索和切换；
  - 会话标题生成；
  - PRD 规定的生成、停止、失败和重试交互；
  - 移动端浏览器适配；
- PC Web 管理端：
  - 使用初始化账号 `admin/admin` 登录；
  - 查看、编辑和保存 OpenAI API Key 与模型；
  - 可选的测试连接；
  - 按话题查询全部用户会话；
  - 查看聊天记录及每次 OpenAI 调用实际送入模型的 Prompt 和实际返回内容；
- 支撑上述功能所需的服务端、数据持久化、OpenAI 集成和访问控制；
- 自动化测试或可重复执行的测试方案，以及关键主线的验收记录；
- 适用于腾讯云服务器的部署配置、环境变量示例、初始化方式、启动方式、更新方式和回滚说明；
- 在用户提供腾讯云访问权限及必要外部配置后，完成实际服务器部署和公网访问验证。

### 不包含

- 真实短信验证码生成和发送；
- 密码登录、第三方登录、游客模式；
- 图片、文件、语音等多模态能力；
- 联网搜索、知识库/RAG、插件和 Agent 工具调用；
- 用户计费、充值、配额、用量和预算管理；
- 多个 LLM 服务商或多套并行生效的模型配置；
- 用户端对话分享、导出、收藏、重命名和删除；
- 管理端用户管理、复杂权限、审计记录和数据分析看板；
- 内容安全、AI 免责声明和内容反馈能力；
- 微信内置浏览器专项适配；
- PRD 明确列入“明确不进入 MVP”的其他功能。

## 任务约束

- 使用 `coordinate-agents-with-local-files` 文件协议协调所有一级 Agent；
- 人工指定的共享本地目录是一级 Agent 之间唯一的实时权威协作空间，正式状态以其中的 Mission、Work Item、Agent、Run、Message 和 Artifact 文件为准；
- GitHub/Git 仅由 Coordinator 作为单一发布者执行阶段性归档和备份，不作为 Message 投递、Polling、状态判定或 Agent 间同步的前提；
- 不使用 GitHub Issue、Pull Request、评论、Projects、Actions、标签或 Webhook 协调工作；
- 所有一级 Agent 访问同一共享本地目录并遵守协议规定的单路径写入权；除 Coordinator 的归档动作外，其他一级 Agent 不执行 Git add、commit、pull、merge 或 push；
- 需要某个一级 Agent 立即关注时，先发布不可变 Message 文件，再通过 Codex 任务通知发送 message_id 和相对路径；任务通知不是正式事实，通知失败时由 Polling 兜底；
- Mission Owner 的一级 `agent_id` 为 `mission-owner`；
- Coordinator 的一级 `agent_id` 为 `coordinator`，由用户单独启动；
- 一级 Agent 可以使用 Sub-agent 完成内部并行工作，但 Sub-agent 不登记到共享协议、不直接 Polling 或收发协议 Message；
- 产品行为以 PRD V1.7 为准；发现 PRD 与实现存在实质冲突时，通过协议 Message 请求 Mission Owner 决策，不得自行扩大或改变产品范围；
- 技术方案由相应 Work Item 产出并接受审核，不在 Mission 中预先锁定具体框架、数据库或部署组件；
- 本期不设置性能指标、容量基线或性能验收门槛；
- 仓库中不得提交 SSH 私钥、OpenAI API Key、腾讯云密钥、服务器密码、Token、生产环境变量或其他凭据；
- 生产凭据只通过人工授权的安全方式提供给确有需要的一级 Agent；
- 服务器、域名、备案、HTTPS 证书、OpenAI API Key 等外部依赖缺失时，执行 Agent 应通过协议 Message 说明最小所需信息；
- 不覆盖或删除用户已有文件；涉及既有内容的修改必须保持可追溯；
- 所有正式交付必须通过 Coordinator 审核并形成 Artifact，未审核的 Run 不作为 Mission 正式成果。

## 必需交付物

- 可运行的 LittleDuck MVP 完整源代码；
- 用户端 H5 与 PRD/UI 参考稿相符的核心页面和交互；
- PC Web 管理端及其核心功能；
- 服务端、数据持久化和 OpenAI 调用能力；
- 数据或应用初始化方案，包括管理员初始账号 `admin/admin`；
- 不包含真实密钥的环境变量示例和配置说明；
- 本地开发、构建、测试和运行文档；
- 关键用户主线和管理主线的测试结果；
- 腾讯云部署所需配置和部署文档；
- 腾讯云服务器上的实际部署结果；
- 用户端和管理端的公网访问入口；
- 服务健康检查、日志查看、进程重启、版本更新和回滚说明；
- 按文件协议形成的 Work Items、Runs、Messages 和经审核接受的 Artifacts；
- Coordinator 汇总的 Mission 交付索引和验收请求。

## 验收标准

- 评审通过的 PRD V1.7 中 P0 功能和 MVP 完成定义均已实现；
- 新用户能够使用手机号和 `000000` 完成注册并进入聊天页；
- 已注册用户能够使用手机号和 `000000` 登录；
- 登录用户能够完成纯文本、多轮、流式 LLM 对话；
- 用户能够创建会话，并在刷新或再次登录后查看、搜索和切换自己的历史会话；
- 普通用户不能查看其他用户会话或管理端内容；
- 管理员能够使用 `admin/admin` 登录独立 PC Web 管理端；
- 管理员能够明文查看、编辑和保存 OpenAI API Key 与模型，且保存不依赖测试连接；
- 管理员能够按话题查看完整聊天记录，以及聊天回复、会话标题生成和重试等每次 OpenAI 调用的实际 Prompt 和返回内容；
- 用户端附件和图片入口已隐藏；
- 关键主线测试通过，并有可复核的测试记录；
- 仓库不包含生产密钥、Token、密码文件或 SSH 私钥；
- 按部署文档可在干净环境中完成构建和启动；
- 腾讯云上的用户端和管理端可通过最终交付的公网地址访问；
- 腾讯云部署使用正确的生产配置后，可完成一次真实 OpenAI 对话；
- 服务器重启后服务能够按部署说明恢复；
- Coordinator 确认所有必需 Work Items 已完成，并且正式 Artifacts 覆盖全部必需交付物；
- Mission Owner 根据公网功能、测试材料、部署材料和 Artifact 索引完成最终验收。
