# 补齐用户端与管理端 UX/UI 规格

## 任务说明

以 PRD V1.7、`/页面UI稿/用户端UI` 的 5 张用户端稿和 `/页面UI稿/管理端UI` 的 6 张管理端稿为依据，重新形成用户端 H5 与独立 PC Web 管理端可开发、可验收的完整高保真页面与交互规格。

## 重新执行说明

- 新增的 6 张管理端正式设计稿推翻了本 Work Item 前一轮交付的视觉基础；revision 5 接受的 Artifact 已撤销，不再供下游使用；
- product-ui-designer 在原 `RUN-WI-002-product-ui-designer` 中重新执行、更新并提交审核，不创建 WI-011 Run；
- 新交付必须完整独立，不只提交与前一轮的差异，也不得要求下游读取已撤销 Artifact 或未审核 Run；
- 保留前一轮已确认的状态、文案、软键盘、安全区、抽屉、滚动和响应式规则，但视觉结构以新增正式设计稿为主要基准。

## 必需交付

- 用户端注册、登录、聊天和历史侧边栏的完整状态清单；
- 375 px 至 430 px H5 布局、软键盘、安全区、滚动和抽屉交互规格；
- 管理员登录、LLM 配置、话题列表、聊天详情和 LLM 调用详情的 PC 页面规格；
- 关键组件、文案、状态迁移和响应式规则；
- 用户端与管理端高保真视觉板、可检查的 HTML/源文件和 PNG 渲染结果；
- 11 张正式 UI 输入到最终页面、组件和状态的映射清单；
- PRD 例外与设计稿取舍清单，以及可重复执行的渲染、内容一致性和凭据模式自测记录。

## 验收标准

- 5 张用户端稿和 6 张管理端稿均有明确映射，无遗漏页面；
- 页面结构、Logo、颜色、字号、间距、控件和信息层级与正式设计稿保持高保真，所有偏差都有 PRD 冲突或缺失状态依据；
- 覆盖 PRD 明确要求的默认、加载、成功、空态、失败、停止和重试状态；
- 用户端隐藏图片、附件、加号和其他多模态入口；
- 管理端适合 1280 px 及以上桌面浏览器并支持长文本、代码块、折叠和复制；
- 话题列表默认每页 20 条，不采用稿中的 10 条；API Key 完整显示明确为假的占位值，不采用掩码；
- 不存在 System Prompt 的调用不补造 system 角色内容；排除编辑、删除、下载、更多菜单等范围外能力；
- HTML/源文件与 PNG 渲染结果内容一致，用户端适配 375 px 至 430 px；
- 交互规格足以让前端实现并让 QA 编写验收用例；
- 不新增 PRD 范围外的用户功能或管理能力；
- 交付中不包含真实 SSH 私钥、API Key、Token、服务器密码或其他凭据。

## 自测要求

- 对全部视觉板执行渲染并检查尺寸、裁切、溢出、字体层级和主要交互状态；
- 逐项核对 11 张输入映射、PRD 例外、完整状态覆盖和 HTML/PNG 一致性；
- 执行凭据模式扫描；发现疑似真实凭据必须移除后才能提交审核；
- 在 Run 的 `answer.md` 汇总自测环境、步骤、结果和已知限制，必要证据放入 `supporting-files/`。
