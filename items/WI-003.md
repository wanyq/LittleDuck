# 实现服务端、数据持久化与 OpenAI 集成

## 任务说明

依据已接受的架构与 API 合同 Artifact，实现 LittleDuck MVP 服务端并实际提供用户端和管理端所需 API。范围包括用户和管理员认证、会话与消息、LLM 配置、流式聊天、停止和重试、标题生成、管理端查询、权限校验及每次 OpenAI 调用的真实记录。

## 必需交付

- 可运行的服务端源代码和数据库迁移或初始化脚本；
- 与当前已接受 API 合同一致的全部 HTTP API、流式接口和错误响应实现；
- 可执行的 API 合同一致性校验、接口测试或等价验证；
- 用户注册、登录、退出和 7 天登录态能力；
- 会话、消息、标题、搜索、分页和访问控制能力；
- OpenAI 配置保存、可选测试连接、聊天流式调用、停止、失败和重试；
- 聊天回复、标题生成和重试的 Prompt、返回、模型、状态及错误记录；
- 管理员登录、配置和话题审查所需接口；
- 自动化测试或可重复执行的服务端验证脚本及说明。

## 验收标准

- 注册和登录仅接受固定验证码 `000000`，且不依赖短信服务；
- 普通用户只能访问自己的会话，管理员接口与用户登录态隔离；
- 同一页面只产生一个有效生成任务，重复提交不会保存重复用户消息；
- 停止、失败和重试保留 PRD 要求的消息与调用状态；
- 每次业务 OpenAI 调用记录实际 Prompt 和实际返回，不伪造不存在的内容；
- 测试连接不写入用户话题调用记录，保存配置不依赖测试成功；
- 管理员初始账号 `admin/admin` 可通过初始化方案获得；
- 实际接口的路径、字段、状态码、错误码和流式事件与当前已接受合同一致；
- 不提交任何真实凭据。

## 前后端协作

- 本 Work Item 负责 API 的实际服务端实现；API 的初始合同由 WI-001 Artifact 提供；
- 可在未完成全部后端逻辑前先提供符合合同的 Mock、Stub 或最小可用接口，但正式交付必须实现真实业务；
- 发现合同不清楚时，通过 `question_asked` Message 同时发送给 Coordinator 和受影响前端 Agent；
- 发现确需改变合同语义时，不得直接修改并要求前端跟随；必须向 Coordinator 发送 `work_item_proposed` Request；
- 只是后端实现偏离合同的，直接在本 Work Item 的同一 Run 中修正，不发起合同变更；
- 若 Coordinator 接受新的合同修订 Artifact，本 Work Item 以最新 revision 中明确引用的合同为准。
