# 实现用户端 H5

## 任务说明

依据 PRD、现有 UI 参考稿、已接受架构、API 合同和 UX/UI Artifact，实现移动端浏览器用户端 H5，包括注册、登录、聊天、流式生成、停止与重试、历史会话、标题搜索和退出登录。

## 必需交付

- 可构建的用户端 H5 源代码；
- 注册页、登录页、聊天页和历史会话侧边栏；
- 流式回复、停止、失败、重试、Markdown 和代码块展示；
- 会话时间分组、搜索、分页、切换和新对话空态；
- 登录失效、网络异常、加载、空态及其他 PRD 状态；
- 移动端适配说明和可重复执行的前端检查。
- 基于已接受 API 合同的客户端封装、Mock 或合同测试，以及与真实后端联调所需说明。

## 验收标准

- 与参考稿的核心布局、品牌和视觉方向一致；
- 在 375 px 至 430 px 宽度无横向滚动，兼容安全区和软键盘；
- 首条消息成功前不创建会话，生成中不能切换会话或再次发送；
- 停止和失败后的重试不重复插入用户消息；
- 历史会话按 Asia/Shanghai 分组并支持标题搜索；
- 用户端不出现图片、文件或其他多模态入口；
- 刷新和重新登录后可恢复已保存内容；
- 请求、响应、错误处理和流式事件消费与当前已接受 API 合同一致；
- 不暴露模型服务内部错误或任何凭据。

## 前后端协作

- 本 Work Item 可基于 WI-001 的已接受合同 Artifact 和 Mock 与 WI-003 并行开发，不读取未审核的后端 Run 作为接口依据；
- 合同含义不清楚时，通过 `question_asked` Message 同时发送给 Coordinator 和后端 Agent；
- 确需新增或改变路径、字段、状态码、错误码、认证、分页或流式事件时，向 Coordinator 发送 `work_item_proposed` Request，不直接要求后端改动；
- 只是前端实现偏离合同的，在本 Work Item 的同一 Run 中修正；
- 若 Coordinator 接受新的合同修订 Artifact，本 Work Item 以最新 revision 中明确引用的合同为准。
