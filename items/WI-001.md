# 确定技术架构并建立工程骨架

## 任务说明

依据 Mission 与 PRD V1.7，形成可实施的系统架构，并建立供后续开发使用的工程骨架。方案需覆盖用户端 H5、PC Web 管理端、服务端、数据持久化、OpenAI 流式调用、身份与权限隔离、调用记录、初始化、测试和腾讯云部署边界，但不得扩大 MVP 范围。

## 必需交付

- 一份架构决策与模块边界说明；
- 数据模型、关键状态、接口和流式交互设计；
- 用户端与管理员身份隔离、会话越权防护和凭据处理方案；
- OpenAI 聊天、标题生成、重试及实际 Prompt/返回记录方案；
- 可启动的工程骨架、无真实密钥的环境变量示例和最小运行说明；
- 对后续开发 Work Item 的明确接口约束。

## 验收标准

- 覆盖 PRD P0 功能及 Mission 的部署目标；
- 技术选型明确且适合在单台腾讯云服务器部署；
- 数据结构能够表达用户、会话、消息、LLM 配置和每次 LLM 调用；
- 明确 SSE 或等价流式机制、停止生成、失败和重试的状态处理；
- 普通用户与管理员的认证、授权和数据访问边界可检查；
- 工程骨架可在无生产凭据的本地环境完成安装或最小启动检查；
- 不包含 API Key、SSH 私钥、Token、服务器密码或其他凭据。
