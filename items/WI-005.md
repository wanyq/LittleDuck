# 实现 PC Web 管理端

## 任务说明

依据 PRD、已接受架构、API 合同和 UX/UI Artifact，实现独立 PC Web 管理端，包括管理员登录、OpenAI 配置、可选测试连接、话题查询、聊天记录和每次 LLM 调用详情。

## 必需交付

- 可构建的 PC Web 管理端源代码；
- 管理员登录、退出和访问控制页面；
- LLM API Key 与模型的查看、编辑、测试连接和保存页面；
- 话题列表的标题、手机号、日期筛选和分页；
- 聊天记录与 LLM 调用详情页面；
- 加载、空态、成功、失败和长文本展示状态；
- 可重复执行的前端检查及运行说明；
- 基于已接受 API 合同的客户端封装、Mock 数据或 Mock 服务、合同测试，以及与真实后端联调所需说明；
- 本次开发自测记录，包含执行环境、命令或步骤、覆盖场景、实际结果、失败项及处理结论。

## 验收标准

- 使用 `admin/admin` 可登录，普通用户登录态不能访问管理端；
- API Key 可明文查看和编辑，保存不依赖测试连接；
- 话题列表默认按最近消息时间倒序，并支持 PRD 要求的筛选；
- 调用详情按时间展示实际 Prompt、返回、服务商、模型、状态和错误；
- 长文本、代码块和结构化内容支持滚动、折叠或复制；
- 页面适合主流桌面 Chrome、Edge 和 1280 px 及以上布局；
- 请求、响应、错误处理、筛选和分页行为与当前已接受 API 合同一致；
- 使用 Mock 数据或 Mock 服务完成本 Work Item 范围内的页面、交互、异常状态和 API 客户端自测，结果全部通过或明确列出不阻断交付的已知限制；
- Mock 的字段、状态码、错误响应、筛选和分页语义与当前已接受 API 合同一致，不以随意构造的数据掩盖合同缺口；
- 不提供用户管理、审计、统计看板或其他范围外功能。

## 自测边界

- 允许且要求在真实后端尚未可用时使用 Mock 数据或 Mock 服务完成本 Work Item 自测；
- 自测至少覆盖管理员登录和退出、LLM 配置查看与编辑、测试连接、保存、话题筛选和分页、聊天记录、LLM 调用详情，以及对应加载、空态和失败状态；
- 提交审核时必须在 Run 的 `answer.md` 中总结自测结论，并把必要的测试输出、截图或报告放入 `supporting-files/`；
- Mock 自测证明管理端实现符合 UI、交互和 API 合同，但不代表真实前后端联调完成；
- 真实后端联调、合同一致性和完整主线验证仍由 WI-006 承担。

## 前后端协作

- 本 Work Item 可基于 WI-001 的已接受合同 Artifact 和 Mock 与 WI-003 并行开发，不读取未审核的后端 Run 作为接口依据；
- 合同含义不清楚时，通过 `question_asked` Message 同时发送给 Coordinator 和后端 Agent；
- 确需新增或改变路径、字段、状态码、错误码、认证、筛选或分页语义时，向 Coordinator 发送 `work_item_proposed` Request，不直接要求后端改动；
- 只是管理端实现偏离合同的，在本 Work Item 的同一 Run 中修正；
- 若 Coordinator 接受新的合同修订 Artifact，本 Work Item 以最新 revision 中明确引用的合同为准。
