# 完成发布稳定化并冻结部署候选

## 任务说明

基于 WI-007 验收 Artifact 和 WI-008 部署 Artifact，修复所有阻断 Mission 验收的缺陷，合并开发期间已接受的必要修订，执行针对性回归，并形成一个与部署材料一致、带唯一版本标识的可部署发布候选。

## 必需交付

- WI-007 缺陷清单的逐项处理结论和代码修复；
- 修复后的完整应用源代码；
- 针对每个阻断缺陷的复测证据；
- 注册登录、聊天、历史会话、管理端和权限隔离关键主线的回归结果；
- 最终 API 合同一致性、构建和凭据扫描结果；
- 与最终代码一致的部署配置及必要文档修订；
- 唯一发布候选版本标识、构建产物说明和变更摘要；
- 一份明确的可部署或不可部署结论。

## 验收标准

- WI-007 识别的所有阻断 Mission 验收缺陷均已关闭并复测通过；
- 非阻断缺陷均有明确处理结论，未修复项不会阻断 Mission 验收；
- 用户端、管理端、服务端和 API 合同保持一致；
- 关键用户主线、管理主线、异常处理和权限隔离回归通过；
- 部署配置和运维文档与最终候选版本保持一致；
- 候选版本可按 WI-008 的部署方式完成构建、初始化和启动；
- 发布候选具有唯一且可复核的版本标识；
- 冻结后不再加入未经新 Work Item 审核的代码或配置修改；
- 不包含 SSH 私钥、API Key、Token、服务器密码或其他生产凭据。
