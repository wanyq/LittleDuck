# 基于新增正式 UI 稿刷新高保真 UX/UI 规格

> 已取消：Mission Owner 确认本工作属于 WI-002 的重新执行，并推翻原交付；相关要求已合并到 WI-002 revision 6。

## 任务说明

以已接受的 WI-002 Artifact 为完整状态与交互基线，并以仓库 `/页面UI稿/用户端UI` 的 5 张用户端稿、`/页面UI稿/管理端UI` 的 6 张管理端稿为主要视觉基准，重新形成可供用户端 H5、PC Web 管理端和 QA 直接消费的完整高保真 UX/UI 规格。

本 Work Item 是 WI-002 的独立后续修订，不修改已完成的 WI-002 Run 或 Artifact。通过后，WI-004 与 WI-005 应优先消费本 Work Item 的 Artifact；本 Artifact 必须能够独立使用，不得要求下游读取未审核 Run。

## 视觉基准与 PRD 例外

- 保持正式设计稿的布局、Logo、颜色、字号、间距、控件和信息层级；只有 PRD 冲突或设计稿缺失必要状态时才修改或补充；
- 用户端隐藏图片、附件、加号及其他多模态入口；
- 话题列表默认每页 20 条，不采用设计稿中的 10 条每页；
- API Key 使用完整明文展示，但视觉稿和示例必须使用明确为假的完整占位值，不采用掩码；
- 不存在 System Prompt 的调用不得补造 system 角色内容；
- 排除编辑、删除、下载、更多菜单及其他 PRD 范围外管理能力。

## 必需交付

- `answer.md`：交付索引、关键结论、与 WI-002 的继承关系及自测结果；
- 覆盖用户端 H5 和 PC 管理端的完整 UX/UI 规格，不只提交差异说明；
- 用户端与管理端高保真视觉板及可检查的 HTML/源文件和 PNG 渲染结果；

- 保留 WI-002 已接受的加载、成功、空态、失败、生成、停止、重试、软键盘、安全区、抽屉、滚动、响应式、长文本、代码块、折叠和复制规则；
- 可重复执行的视觉尺寸、内容一致性和凭据模式自检记录。

## 验收标准

- 5 张用户端稿和 6 张管理端稿均有明确映射，无遗漏页面；
- 关键页面的结构、品牌、颜色、字号、间距、控件和信息层级与正式设计稿保持高保真，所有偏差都有 PRD 或缺失状态依据；
- 用户端与管理端均覆盖 WI-002 已接受的完整状态和交互规则，且没有引入范围外功能；
- 用户端不出现图片、附件、加号或其他多模态入口；
- 管理端默认每页 20 条，API Key 完整显示明确为假的占位值，不补造不存在的 System Prompt；
- HTML/源文件与 PNG 渲染结果内容一致，用户端适配 375 px 至 430 px，管理端适配 1280 px 及以上；
- 规格足以让 WI-004、WI-005 实现并让 QA 编写验收用例；
- 交付中不包含真实 SSH 私钥、API Key、Token、服务器密码或其他凭据。
