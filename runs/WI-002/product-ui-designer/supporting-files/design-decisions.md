# PRD 例外与设计稿取舍

版本：WI-002 revision 6

## 1. 决策优先级

1. 功能、数据、权限、状态与验收以 PRD V1.7 为准。
2. 在不冲突时，页面结构、Logo、颜色、字号、间距、圆角、阴影、控件形态和信息层级以 11 张正式 UI 稿为准。
3. 正式稿未覆盖的状态，复用 `组件补充稿.png` 的骨架、空态、失败、按钮、校验、Toast、折叠与复制语言；H5 缺失状态沿用用户端绿色、白色与浅灰视觉体系。
4. 不通过视觉补稿引入 PRD 范围外的功能。

## 2. 已确认偏差

| 决策 | 正式稿表现 | 最终规格 | 依据与影响 |
| --- | --- | --- | --- |
| H5 多模态入口 | 聊天默认稿有“+”，输入态有图片图标 | 两者均从最终输入区移除；仅文本框 + 发送/停止 | PRD 4.2、8.3、14.9 明确图片、附件、语音不在 MVP；不改变气泡、导航和输入胶囊方向 |
| 注册/登录验证码按钮位置 | 注册稿位于手机号行，登录稿位于验证码行 | 分别保留稿件布局；交互规则一致 | PRD 未要求二页按钮必须同位；保留已有视觉稿优先 |
| H5 固定高画布 | 稿件为 1088 × 1936 静态图 | 以 375–430 CSS px、`100dvh` 和安全区实现；比例而非绝对像素还原 | PRD 10.3 要求软键盘、刘海、横竖屏和动态视口可用 |
| 侧边栏右上菜单 | 输入稿遮罩后的聊天页仍隐约可见菜单图形 | 抽屉打开后底层完全不可点击；不实现更多菜单 | PRD 8.4 只允许范围内关闭方式，且禁止重命名、删除、置顶 |
| 管理端标题用词 | 登录稿使用“LittleDuck 管理后台”，左侧品牌分两行 | 视觉品牌按稿保留；规格统一称“管理端”作为产品端类型 | 仅文案呈现差异，不改变功能或登录隔离 |
| 话题列表分页 | 稿件显示 `10 条/页` | 默认 `20 条/页` | WI-002 revision 6 与 PRD 9.3.2 明确 20 条；视觉仍沿用原分页控件 |
| 话题表格操作列 | 稿件有“操作 / 查看详情” | 允许唯一只读动作“查看详情”；标题本身也可进入详情 | PRD 允许查看详情，排除编辑、删除和批量操作 |
| API Key 示例 | 配置稿显示类似真实格式的短示例；组件稿展示掩码 | 最终板仅使用完整明文假值 `EXAMPLE_NOT_A_REAL_OPENAI_API_KEY_000000`；不提供显隐切换 | PRD 9.2.3 要求完整明文；WI-002 验收要求明确假值且不采用掩码 |
| System Prompt 行 | 调用详情稿示例中有 system 行 | 角色行完全数据驱动；不存在时不渲染，不创建空占位，即不补造 System Prompt | PRD 9.3.4 要求真实角色与顺序，明确不得补造不存在内容 |
| 组件补充稿能力图标 | 样例列出编辑、删除、查看、复制、下载、更多 | 页面仅采用复制、范围内查看和状态徽标；排除编辑、删除、下载、更多 | PRD 2.2、4.2、9.3 明确只读且不提供导出/删除等能力 |
| 长文本默认收起 | 组件稿给出展开/收起 | 聊天正文 240 px、代码块 320 px、Prompt/返回 360 px 后折叠，支持展开、内部滚动与复制 | PRD 9.3.3–9.3.4 和 10.3 要求长文本、代码块、折叠及复制 |
| 管理端窄屏 | 正式稿基准为 1440 × 1024 | 1280 px 以上正常布局；1180–1279 保留桌面结构并横向滚动；不做手机重排 | PRD 9、10.3 明确为独立 PC Web，不复用 H5 |

## 3. 未新增能力声明

最终规格不包含会话重命名、删除、置顶、分享、导出、收藏、成功回复重新生成、多模型切换、System Prompt 配置、管理端用户管理、用量看板、Prompt 编辑、管理员重发用户对话、配置版本、回滚或审计。

## 4. 视觉一致性判定

- 正式页面：继承对应设计稿的框架、Logo、白/浅灰底、绿色强调、低阴影白卡和信息层级。
- 最终补稿不再用 CSS 圆环、白色椭圆或橙色块拼成占位 Logo，也不跨端合并正式图形：H5 注册/导航/欢迎/助手头像/抽屉只用正脸 `littleduck-logo.svg`；管理端登录/侧栏只用整鸭 `littleduck-admin-logo.svg`。
- 缺失状态：必须看起来属于同一页面，不用全屏弹窗替换原有上下文；失败时保留可恢复入口与已成功数据。
- 所有状态同时使用文字/图标，不只依赖颜色。
- 视觉板内的正式稿缩略图仅作为输入映射证据；其上标出的 PRD 冲突不代表最终实现。
